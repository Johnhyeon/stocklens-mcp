# StockLens 품질 검증 기록

릴리즈 직전 실행하는 자동 QA·부하 테스트 결과 모음입니다. 개발 세션이 유지 관리합니다.

---

## 📋 검증 범위

| 카테고리 | 항목 |
|---------|------|
| **기능 QA** | 정상 케이스 + 엣지 케이스 (빈 입력·잘못된 코드·국제 심볼·ETF·REIT·BDC·페니주·BRK.B 같은 dot ticker) |
| **의심 패턴 스캔** | `None` leak, `nan`/`inf`, `NaT`, HTML 태그, 더블 스페이스, unix timestamp 노출, 비정상 퍼센트 |
| **응답 크기** | 대용량 차트(`max, 1d`) / 장기 분봉 / 필링 50개 등 토큰 한계 검사 |
| **콜드 스타트** | 서버 기동 시 import 비용 + 첫 tool 호출 latency |
| **동시성 / 스트레스** | 20·50·100 티커 동시 호출 시 rate limit 여부 |
| **장시간 Soak** | 30분 연속 호출 시 RSS·FD·캐시 증가 추이 (누수 감지) |
| **캐시** | TTL 만료 후 재호출 검증, 캐시 키 분리 |
| **네트워크 복원** | 타임아웃·연결 실패 시 사용자 메시지 친화도 |

## ✅ 결과 요약 (v0.3.0 릴리즈 직전)

| 지표 | 결과 |
|------|------|
| **기능 QA 케이스** | 400+ (KR 56 + US 350+), 의심 패턴 **0건** |
| **Soak (30분 연속)** | **64,625 호출 / 오류 0건** (성공률 100%, 43 tool 랜덤 순회) |
| **메모리** | RSS 167 → 467 MB (Δ+300). 환산 4.64 KB/call, 실사용 ~8MB/hr |
| **FD / 연결 풀** | 512 → 658 안정 (t=500s 이후 완전 평탄) |
| **캐시** | 422 entries 포화, TTL 만료·키 분리 동작 확인 |
| **스트레스 (100 동시)** | 100개 티커 동시 `get_price` 3.18s, 429 rate limit 미발생 |
| **콜드 스타트** | Import 1.35s, 첫 tool 호출 0.95s, 캐시 hit 0ms |
| **응답 크기 보호** | `get_us_chart` 500행 상한 적용 (원본 11,428행 → 500행 자동 축약) |
| **네트워크 오류 복원** | Timeout / ConnectionError 시 친화적 메시지 |
| **멀티 티커 병렬화** | 30 티커 일괄 조회 11.2s → **1.3s (8.4×)** |

### 메모리 장기 사용 주의
Soak 30분 기준 RSS가 plateau에 도달하지 않았습니다 (마지막 10분 구간 2-3 MB/min 점증).
- 실사용 환산: 8시간 Claude 세션 ≈ 67MB 누적 → 일상 재시작으로 해소됨
- 무한 연속 세션은 현재 보장 불가 — Claude Desktop 일일 재시작 권장
- 근본 원인 조사(yfinance 내부 캐시 / pandas DataFrame / ThreadPool) 는 post-release 과제

## 🐛 릴리즈 전 수정된 이슈

| # | 이슈 | 원인 | 수정 |
|---|------|------|------|
| 1 | `get_us_news` 호출 시 `NameError` | `import re` 누락 | import 추가 |
| 2 | ETF `get_us_info` 응답에 `None` 문자열 노출 | `dict.get(k, '-')`는 `None` 키일 때 `None` 반환 | `or '-'` 패턴으로 교체 |
| 3 | 기관 보유자 표 **더블 스페이스** | yfinance holder name에 trailing space | `.strip()[:N].rstrip()` |
| 4 | ETF에 analyst/earnings/holders 요청 시 빈 출력 | 데이터 없는데 안내 메시지 없음 | 명시적 안내 추가 |
| 5 | `get_us_sector('TECHNOLOGY')` 대문자 실패 | Yahoo는 소문자 기대 | 입력 정규화 |
| 6 | 무효 섹터 입력 시 애매한 에러 | 어떤 섹터가 유효한지 모름 | 유효 섹터 11개 나열 |
| 7 | ETF expense_ratio / YTD 스케일 오류 | yfinance가 이미 % 단위인데 `*100` 중복 | 로직 교정 |
| 8 | 대용량 차트 토큰 폭주 (`max, 1d` = 728KB) | 무제한 기간 허용 | 500행 상한 자동 자름 |
| 9 | 멀티 티커 30개 조회 11초 | 내부 순차 fetch | `asyncio.gather` 병렬화 |

## 🔍 한계 / 알려진 제약

릴리즈에 **포함된 상태**의 제약입니다. 업그레이드 로드맵 있음.

- **Yahoo Finance 15분 지연** — 실시간 호가·다크풀·Level 2 미지원
- **옵션 Greeks 미제공** — yfinance 자체 제공 안 함. 필요 시 IV+Spot+Strike로 클라이언트 계산
- **BRK.B SEC 공시** — yfinance 데이터 gap (BRK-A로 조회 필요)
- **공매도 2~4주 stale** — FINRA bi-monthly 공시 스케줄 제약, 응답에 `date_short_interest` 표시
- **한국 주식 네이버 HTML 변동 리스크** — 스크레이핑 구조 변경 시 일부 필드 파싱 실패 가능. 릴리즈마다 재검
